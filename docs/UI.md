## 2. 核心 Demo 工作区（重点）

### 2.1 整体布局

```
┌─────────────────────────────────────────────┐
│  工作台标题 + "实时生成"徽章                    │
├──────────────────┬──────────────────────────┤
│                  │                          │
│   输入面板        │      输出面板             │
│   (左, 1fr)      │      (右, 1.1fr)         │
│                  │                          │
└──────────────────┴──────────────────────────┘
```

- 桌面端：左右分栏，中间竖线分隔
- 移动端：上下堆叠，输入在上、输出在下

---

### 2.2 输入面板

#### 2.2.1 场景选择（新增，现有 HTML 没有）

在输入框上方提供 3 个场景选项卡（单选）：

| 场景 | 作用 | 对应示例内容方向 |
|------|------|------------------|
| 会议纪要 | 默认选中，结构化会议记录 | 周会、评审会纪要 |
| 群聊记录 | 群聊讨论内容 | 微信/飞书群消息粘贴 |
| 老师或领导布置任务 | 任务布置内容 | 导师/领导口头或文字任务 |

**交互**：
- 单选样式，选中项绿色填充，未选中白底灰边
- 切换场景时：清空当前输入框，更新 placeholder 文案，更新"填入示例"按钮对应的示例内容
- 场景选择会随输入一起传给后端，用于选择不同 Prompt 模板（后端实现，UI 只负责传值）

#### 2.2.2 输入区

| 元素 | 说明 |
|------|------|
| 标签 | "会议内容输入区"（随场景切换文案，如"群聊记录输入区"） |
| 工具按钮 | "填入示例" / "清空" |
| 文本框 | 多行 textarea，最小高度 280px，可垂直拉伸 |
| placeholder | 随场景变化，给出格式提示 |
| 隐私提示 | 文本框下方小字："请勿粘贴含敏感商业信息或个人隐私的内容" |
| 字数统计 | 文本框右下角实时显示"当前字数 / 2000" |

**输入限制**：
- 软限制 2000 字，超过时字数变橙色
- 硬限制 3000 字，超过禁止继续输入并提示（见 3.6 超长输入状态）

#### 2.2.3 按钮组

| 按钮 | 样式 | 行为 |
|------|------|------|
| 填入示例 | 次按钮（白底灰边） | 根据当前场景填入对应示例文本 |
| 清空 | 次按钮（白底灰边） | 清空输入框 + 输出区，回到空状态 |
| 生成行动方案 | 主按钮（绿色填充） | 触发 AI 生成，加载时显示 loading + 禁用 |

**生成按钮状态**：
- 默认：绿色，文字"⚡ 生成行动方案"
- 加载中：显示旋转 spinner，文字隐藏，按钮禁用
- 禁用（输入为空）：灰色，不可点击

### 2.3 输出面板

#### 2.3.1 Tab 切换

4 个 Tab 横向排列，单选：

| Tab | 图标 | 文案 |
|-----|------|------|
| 任务清单 | 📋 | 任务清单 |
| 风险提醒 | ⚠️ | 风险提醒 |
| 跟进消息 | 💬 | 跟进消息 |
| 汇报草稿 | 📊 | 汇报草稿 |

- 选中项：白底 + 绿色文字 + 阴影
- 未选中：灰字，hover 变深
- 默认选中"任务清单"
- 移动端：4 个 Tab 等宽排列，文字可能需要缩短（任务/风险/消息/汇报），确保不溢出

#### 2.3.2 Tab 1：任务清单

每条任务卡片字段：

| 字段 | 样式 | 说明 |
|------|------|------|
| 编号 | 绿色圆角方块，等宽字体 | 1, 2, 3... |
| 任务描述 | 主文字，深色 | 任务内容（已 HTML 转义） |
| 负责人 | 蓝色标签 👤 | 人名或"未指派"（红色） |
| 截止时间 | 黄色标签 ⏰ | 时间或"未指定"（橙色） |
| 优先级 | 高=红 / 中=黄 / 低=绿 | 🔴高 🟡中 🟢低 |
| 依赖 | 灰色标签 🔗 | "依赖 XX"（如有） |
| 待确认 | 灰色标签 ❓ | 需求不清标记（如有） |

**空数据**：显示空状态图标 + "未识别到任务，请尝试粘贴更详细的会议内容"

#### 2.3.3 Tab 2：风险提醒

4 类风险卡片，按级别配色：

| 类型 | 图标 | 标题颜色 | 背景色 | 触发条件 |
|------|------|----------|--------|----------|
| danger | ⚠️ | 红 | 浅红 | 缺少负责人 |
| warning | ⏰ / ❓ | 黄 | 浅黄 | 缺截止时间 / 需求不清 |
| info | 🔗 | 蓝 | 浅蓝 | 依赖阻塞 |

每条风险含：图标 + 标题 + 描述（描述中任务名截断到 24 字 + "..."）。

**无风险时**：显示 ✅ + "暂无风险，当前任务清单未检测到明显风险，建议定期跟进进度。"

#### 2.3.4 Tab 3：跟进消息

纯文本草稿，结构：

```
📋 会后行动项跟进
━━━━━━━━━━━━━━━

@张明
  【高优】完成新版首页设计稿（截止：本周五）
  【常规】准备客户演示PPT（截止：下周三）

@李娜
  【常规】跟进用户调研报告（截止：下周一）

⚠️ 以下任务尚未指派负责人：
  · 营销活动方案

🔴 需关注的风险：
  · 依赖阻塞：后端接口开发依赖赵雪的产出

请各位确认任务和截止时间，有问题及时同步。
```

**渲染规则**：
- `@人名` 高亮绿色加粗
- `【标签】` 高亮橙色加粗
- 分隔线 `━━━` 灰色
- 整体等宽字体感，保留换行

**操作**：底部"📋 复制消息"按钮，真实写入剪贴板。

#### 2.3.5 Tab 4：汇报草稿

结构化汇报，5 个段落：

| 段落 | 内容 |
|------|------|
| 【本周行动项总览】 | 共 N 项，高优 X 项，中优 Y 项，低优 Z 项 |
| 【高优先级任务】 | 任务 — 负责人，截止 |
| 【常规任务】 | 任务 — 负责人，截止 |
| 【风险项】 | 阻塞/待确认/待分配，带红色标签 |
| 【下一步计划】 | 4 条固定建议 |

**渲染规则**：
- `【段落标题】` 渲染为带下边框的小标题
- 每条 `·` 项渲染为带绿色圆点的列表项
- 风险标签 `<span class="tag risk">` 渲染为红色小标签

**操作**：底部"📋 复制草稿"按钮，真实写入剪贴板。

## 3. 状态设计

### 3.1 空状态（初始）

| 位置 | 表现 |
|------|------|
| 输入框 | 显示 placeholder |
| 输出区 | 居中图标 📝 + "在左侧粘贴会议内容并点击'生成行动方案'，这里将展示结构化的行动方案" |

触发：页面加载 / 点击清空 / 生成成功后切换 Tab 无数据。

---

### 3.2 加载状态

| 元素 | 表现 |
|------|------|
| 生成按钮 | 显示 spinner，隐藏文字，按钮禁用 |
| 输出区 | 显示居中加载动画 + 文案"AI 正在整理行动方案..." |
| 输入框 | 不禁用（允许用户继续编辑，但生成按钮已禁用） |
| Tab | 禁用切换（避免渲染半成品） |

**超时处理**：超过 30 秒自动转为错误状态，提示"AI 响应超时，请重试"。

---

### 3.3 成功状态

| 元素 | 表现 |
|------|------|
| 生成按钮 | 恢复默认，可再次点击 |
| 输出区 | 渲染当前 Tab 内容，带 fadeInUp 动画（每项延迟 0.08s） |
| Tab | 可正常切换 |

**真实 AI 输出标识**：输出区顶部显示小标签"✓ AI 已生成"（绿色），与错误提示明确区分。

---

### 3.4 错误状态（AI 异常）

| 错误类型 | 表现 |
|----------|------|
| AI 接口报错 | 红色提示卡片：图标 ❌ + "生成失败" + 具体原因 + "重试"按钮 |
| AI 返回格式异常 | 橙色提示卡片：图标 ⚠️ + "AI 返回内容无法解析，请重试" |
| 网络错误 | 红色提示卡片：图标 🌐 + "网络连接失败，请检查后重试" |
| 超时 | 红色提示卡片：图标 ⏰ + "AI 响应超时，请重试" |

**错误卡片样式**：浅红背景 + 红边框，与成功状态（绿色"AI 已生成"标签）形成对比。
**错误时不覆盖输入内容**，用户可直接点"重试"。

---

### 3.5 AI Key 未配置状态

触发：后端检测到 AI API Key 未配置，返回特定错误码。

| 位置 | 表现 |
|------|------|
| 输出区 | 黄色提示卡片：图标 🔧 + "AI 服务未配置" + 说明"演示环境的 AI 接口尚未就绪，请联系部署者配置 API Key" |
| 生成按钮 | 禁用或点击后直接显示该提示 |

**注意**：此状态面向 Demo 部署场景，文案要友好，不暴露技术细节（如 Key 名称、环境变量名）。

---

### 3.6 超长输入状态

| 阈值 | 表现 |
|------|------|
| 1500–2000 字 | 字数统计变橙色，提示"内容较长，AI 处理可能稍慢" |
| 2000–3000 字 | 字数统计变红色，提示"已超过建议长度，建议精简后生成" |
| >3000 字 | 禁止继续输入，提示"内容过长，请精简到 3000 字以内" |

---

### 3.7 复制成功 / 复制失败状态

| 状态 | 表现 |
|------|------|
| 复制成功 | 按钮文字变"✓ 已复制"，文字和边框变绿色，2 秒后恢复 |
| 复制失败 | 按钮文字变"✗ 复制失败"，文字变红色，2 秒后恢复；同时可考虑降级用 textarea + execCommand 兜底 |

**关键**：必须真实调用 `navigator.clipboard.writeText()` 写入剪贴板，不能只改按钮文字（见第 6 节修正项）。

## 4. 响应式设计

### 4.1 断点

| 断点 | 范围 | 布局 |
|------|------|------|
| 桌面 | ≥901px | Demo 左右分栏，导航完整 |
| 平板/手机 | ≤900px | Demo 上下堆叠，导航隐藏链接 |

### 4.2 桌面端（≥901px）

- Demo 工作区：左右分栏（输入 1fr : 输出 1.1fr）
- 输入面板右 border 分隔
- 输出区最大高度 440px，内部滚动
- 4 个 Tab 横向等宽

### 4.3 手机端（≤900px）

- Demo 工作区：上下堆叠，输入在上
- 输入面板底部 border 分隔（替代右 border）
- 输出区高度自适应（最小 360px）
- **Tab 处理**：
  - 4 个 Tab 等宽排列，文字缩短为"任务/风险/消息/汇报"
  - Tab 容器不换行，整体可横向滚动（如仍溢出）
  - Tab 点击区域 ≥44px 高（符合触控规范）
- 场景选择：3 个选项横向排列，文字可缩短
- 按钮：生成按钮全宽，填入示例/清空并排

### 4.4 小屏微调（≤560px）

- section padding 缩小到 56px 0
- 容器 padding 缩小到 16px
- Hero 标题字号缩小
- 价值卡片单列

## 5. 视觉风格

### 5.1 色彩系统（沿用现有）

| 变量 | 值 | 用途 |
|------|-----|------|
| --bg | #F5F4EF | 页面底色（暖白） |
| --surface | #FFFFFF | 卡片背景 |
| --ink | #1A1915 | 主文字 |
| --ink-3 | #6B6759 | 次要文字 |
| --primary | #0D6E5C | 主色（绿） |
| --primary-2 | #0A5648 | 主色 hover |
| --accent | #C2620E | 强调色（橙） |
| --danger | #C0392B | 错误/高风险 |
| --warning | #B8860B | 警告/中风险 |
| --success | #2D7A3E | 成功/低风险 |
| --info | #1A7A8C | 信息/依赖 |

**禁止使用蓝紫配色**（遵循设计约束）。

### 5.2 字体（修正：不依赖 Google Fonts）

现有 HTML 用了 Google Fonts（Fraunces / Plus Jakarta Sans / JetBrains Mono），需改为**系统字体栈**，保证离线可打开：

| 用途 | 旧字体 | 新字体栈 |
|------|--------|----------|
| 展示标题 | Fraunces | Georgia, 'Songti SC', 'STSong', 'SimSun', serif |
| 正文 | Plus Jakarta Sans | -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif |
| 等宽 | JetBrains Mono | 'Cascadia Code', 'Consolas', 'Courier New', monospace |

保留"展示衬线 + 正文无衬线 + 等宽"三层字体层次，仅替换来源。

### 5.3 风格基调

| 维度 | 要求 |
|------|------|
| 整体感受 | 工作效率工具，**非营销落地页** |
| 信息密度 | 适中，不过度留白也不过度堆砌 |
| 装饰程度 | 克制，按钮和卡片不加过度阴影/渐变 |
| 动效 | 轻量，仅用于状态切换（fadeInUp、loading spinner），不喧宾夺主 |
| 圆角 | 统一 8px/12px/20px 三级 |
| 阴影 | 三级（sm/md/lg），hover 时轻微上浮 |

**与现有 HTML 的差异**：现有 HTML 偏营销页（大 Hero、起源故事、价值卡片）。MVP 应**弱化营销区，强化 Demo 工作区**，让用户首屏就能看到输入框和生成按钮。

## 6. 需要修正现有 HTML 的 UI 问题

以下 4 项是现有 huihou-xingdongpai.html 必须修正的问题，开发时按此处理：

### 6.1 不依赖 Google Fonts

| 问题 | 现有 HTML 通过 Google Fonts CDN 加载 Fraunces / Plus Jakarta Sans / JetBrains Mono，离线或内网环境无法打开 |
|------|---|
| 修正 | 删除 `<link>` Google Fonts 引用，改用第 5.2 节的系统字体栈 |
| 验收 | 断网后打开 HTML，字体仍正常显示，无 FOUT 闪烁 |

### 6.2 复制按钮必须真实复制内容

| 问题 | 现有 `copyDraft()` 函数只改变按钮文字为"已复制"，**没有实际调用 clipboard API 写入剪贴板** |
|------|---|
| 修正 | 使用 `navigator.clipboard.writeText(text)` 写入剪贴板；失败时降级用临时 textarea + `document.execCommand('copy')` 兜底 |
| 验收 | 点击复制后，在系统剪贴板粘贴，内容与展示一致（对应 PRD A7） |

### 6.3 输出内容必须 HTML 转义

| 问题 | 现有代码用 `innerHTML` 直接拼接 AI/用户内容（如 `task-text`、`risk-desc`），存在 XSS 风险 |
|------|---|
| 修正 | 所有来自用户输入或 AI 返回的文本，渲染前必须经过 `escapeHtml()` 转义（`< > & " '`）；只有受信任的结构标签（如 `<span class="...">`）才允许拼接 |
| 验收 | 输入 `<script>alert(1)</script>` 或 `<img onerror=...>`，输出区原样显示文本，不执行脚本（对应 PRD 8.3） |

### 6.4 区分真实 AI 输出与错误提示

| 问题 | 现有 HTML 是纯前端规则解析，无错误状态；接 AI 后需要明确区分"AI 成功输出"和"错误提示" |
|------|---|
| 修正 | 成功时输出区顶部显示绿色"✓ AI 已生成"标签；错误时显示红/橙色错误卡片（见 3.3 / 3.4）；两者样式强对比，不混淆 |
| 验收 | AI 成功 → 绿色标签 + 内容；AI 失败 → 红色卡片 + 重试按钮；用户能一眼区分 |
