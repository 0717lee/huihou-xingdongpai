# 会后行动派 · PRD（产品需求文档）

> 赛道：TRAE AI 创造力大赛 · 学习工作赛道
> 版本：v1.0（MVP / Demo 版）
> 日期：2026-06-20
> 状态：待评审

---

## 0. 文档说明

本 PRD 以**比赛 Demo 可演示、核心链路可跑通**为第一目标，范围刻意做小而完整。
所有功能围绕一条主链路：**粘贴文本 → AI 生成 → 结构化展示 → 一键复制**。
不在本版本范围内的能力（登录、协作、数据库等）见第 5 节。

---

## 1. 产品背景和痛点

### 1.1 背景
会议本身不是问题，问题是**开完会到真正有人动手之间**的那段"隐性劳动"：
整理纪要、拆任务、确认负责人、补截止时间、写跟进消息、写汇报草稿。
这段劳动没人想做，但又不能不做，最终导致"开会很多，推进很慢"。

### 1.2 核心痛点（4 类）
| 痛点 | 表现 | 后果 |
|------|------|------|
| 纪要散落 | 结论散在文档、群聊、邮件里 | 整理一份完整纪要要 30 分钟+ |
| 责任不清 | "谁来做"靠口头确认 | 事后追溯找不到负责人 |
| 时间模糊 | "尽快""下周再说" | 截止日期形同虚设 |
| 跟进困难 | 无结构化清单和风险提醒 | 下次开会才发现进度卡住 |

### 1.3 一句话定位
**把会议纪要、群聊记录、老师或领导布置的任务粘贴进去，AI 自动整理出任务清单、负责人、截止时间、优先级、风险提醒和汇报草稿。**

---

## 2. 目标用户

面向"会后要推进事情"的人，Demo 聚焦三类典型用户：

| 用户类型 | 角色 | 核心诉求 |
|----------|------|----------|
| 职场项目经理 | PM / 团队负责人 | 周会后快速拆任务、分配负责人、生成周报 |
| 学生团队负责人 | 课题组长 / 社团干部 | 小组讨论后明确分工和截止时间，减少沟通成本 |
| 跨团队协作者 | 产品 / 运营 / 开发 | 需求评审后明确各方职责、识别依赖、避免阻塞 |

**共性特征**：经常开会、需要会后推进、手动整理耗时、对"责任到人、时间明确"有强需求。

---

## 3. 核心使用场景

Demo 需覆盖以下 5 个场景，均通过"粘贴文本 → 生成方案"同一链路完成：

1. **团队周会跟进**：粘贴周会纪要，生成本周任务清单和跟进消息。
2. **项目需求评审**：粘贴评审记录，识别各方职责和前后依赖。
3. **学生小组分工**：粘贴讨论记录，明确分工和截止时间。
4. **导师/领导布置任务**：粘贴任务布置内容（语音转文字亦可），拆解执行项。
5. **跨部门会议同步**：粘贴会议记录，生成统一跟进消息，减少信息差。

## 4. MVP 范围

### 4.1 产品形态
- **单页 Web 应用**（无需下载、无需登录）
- 前端：静态 HTML/CSS/JS
- 后端：轻量 Node.js 服务（仅用于代理 AI 接口、保护 API Key）
- AI：OpenAI 兼容协议（如阿里云 DashScope 兼容模式）

### 4.2 核心链路（必须跑通）
```
用户粘贴文本
    ↓
点击"生成行动方案"
    ↓
前端 → 后端 → AI 接口
    ↓
AI 返回结构化 JSON
    ↓
前端渲染 4 个 Tab：任务清单 / 风险提醒 / 跟进消息 / 汇报草稿
    ↓
用户一键复制所需内容
```

### 4.3 MVP 必做功能
见第 6 节核心功能列表 F1–F8。

---

## 5. 明确不做什么（Out of Scope）

为保障 Demo 聚焦，本版本**明确不做**以下能力：

| 不做项 | 原因 |
|--------|------|
| 用户登录/注册 | Demo 无需身份，降低使用门槛 |
| 团队协作/多用户编辑 | 超出单链路范围 |
| 数据库持久化 | 一次会话一次生成，不存储 |
| 复杂权限管理 | 无用户体系则无需权限 |
| 任务看板/项目管理 | 本产品只做"生成"，不做"管理" |
| 日历/IM 集成 | 不自动发送、不同步日历 |
| 移动端 App / 小程序 | Demo 仅做 Web 响应式 |
| 历史记录/会话存档 | 不存储任何输入内容 |
| 多语言/国际化 | Demo 仅支持中文 |
| 离线模式 | 依赖 AI 接口，必须联网 |

## 7. 输入输出说明

### 7.1 输入
- **形式**：纯文本（用户粘贴）
- **来源**：会议纪要、群聊记录（截图转文字）、任务布置内容（语音转文字）
- **格式**：任意格式，无需预先整理
- **长度限制**：建议 200–2000 字（超出截断提示）
- **示例输入**：
```
产品周会 · 2024年3月15日
参会人员：张明、李娜、王强、赵雪、陈晨

1. 张明负责本周五前完成新版首页设计稿，比较紧急。
2. 李娜跟进用户调研报告，下周一前提交。
3. 王强说后端接口开发需要等赵雪的数据库设计完成才能开始。
4. 营销活动的方案还没定，需要再讨论一下。
5. 赵雪负责数据库设计，本周三前完成。
```

### 7.2 输出（4 类，Tab 切换）

#### Tab 1：任务清单
结构化任务列表，每条任务包含：
- 任务编号
- 任务描述
- 负责人（未识别则标"未指派"）
- 截止时间（未识别则标"未指定"）
- 优先级（高/中/低）
- 依赖标记（如有）
- 待确认标记（如有）

#### Tab 2：风险提醒
自动识别 4 类风险：
| 风险类型 | 触发条件 | 级别 |
|----------|----------|------|
| 缺少负责人 | 任务未识别到负责人 | danger |
| 缺少截止时间 | 任务未识别到截止时间 | warning |
| 依赖阻塞 | 任务存在前置依赖 | info |
| 需求不清晰 | 出现"待定/再讨论/未定"等表述 | warning |

#### Tab 3：跟进消息草稿
- 按负责人分组，`@人名` 形式
- 每条任务标注优先级标签（【高优】/【常规】/【低优】）
- 列出未指派任务
- 列出需关注的风险（最多 3 条）
- 结尾统一提示语

#### Tab 4：汇报草稿
结构化汇报，包含：
- 本周行动项总览（任务数、优先级分布）
- 高优先级任务清单
- 常规任务清单
- 风险项（阻塞/待确认/待分配）
- 下一步计划

### 7.3 AI 返回数据结构（约定）
后端要求 AI 返回如下 JSON 结构，前端按此渲染：
```json
{
  "tasks": [
    {
      "id": 1,
      "task": "完成新版首页设计稿",
      "owner": "张明",
      "deadline": "本周五",
      "priority": "高",
      "dependency": null,
      "unclear": false
    }
  ],
  "risks": [
    { "type": "warning", "icon": "⏰", "title": "缺少截止时间", "desc": "..." }
  ],
  "message": "📋 会后行动项跟进\n...",
  "report": "【本周行动项总览】\n..."
}
```

## 8. 安全与隐私边界

### 8.1 数据安全
| 项 | 策略 |
|----|------|
| 输入内容 | **不持久化、不存储**，生成后即丢弃 |
| 历史记录 | 不保存任何会话历史 |
| 用户信息 | 不收集任何用户身份信息 |
| API Key | **仅在后端持有**，前端不可见，不硬编码 |

### 8.2 隐私提示
- 在输入区显著位置提示：**"请勿粘贴含敏感商业信息或个人隐私的内容"**
- 建议用户对真实人名、公司名、项目名做脱敏后再粘贴

### 8.3 接口安全
- 后端对输入文本做长度限制（防止滥用）
- 后端对 AI 返回做结构校验（防止渲染异常）
- 后端设置简单并发限制（Demo 环境）
- 不开放公开 API，仅前端页面调用

### 8.4 AI 服务边界
- 使用 OpenAI 兼容协议（如阿里云 DashScope 兼容模式）
- 明确告知用户：输出由 AI 生成，可能存在误差，关键决策需人工确认

## 9. 验收标准

### 9.1 核心链路验收（必须全部通过）
| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| A1 | 粘贴示例 → 生成 | 点击“填入示例”后点击“生成”，显示加载状态，并在合理时间内返回结果；AI 超过 30 秒需转为超时错误状态 |
| A2 | AI 真实调用 | 后端日志可见 AI 接口请求与响应，非前端规则模拟 |
| A3 | 任务清单正确 | 至少识别出 5 条任务，含负责人、截止时间、优先级 |
| A4 | 风险提醒正确 | 至少识别出 2 条风险（如依赖阻塞、需求不清） |
| A5 | 跟进消息可用 | 消息含 @人名、优先级标签、风险项，可直接复制发送 |
| A6 | 汇报草稿可用 | 含总览/高优/常规/风险/下一步 5 个段落 |
| A7 | 一键复制 | 复制按钮必须真实写入剪贴板（非仅改变按钮文字），剪贴板内容与展示一致 |

### 9.2 健壮性验收
| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| B1 | 空输入 | 点击生成时提示"请先粘贴会议内容"，不报错 |
| B2 | 超长输入 | 超过 2000 字时提示截断或拒绝，不卡死 |
| B3 | AI 异常 | AI 接口超时/报错时，前端展示友好错误提示，不白屏 |
| B4 | 非会议内容 | 粘贴无关内容时，AI 仍能返回结构（可为空任务+提示） |

### 9.3 体验验收
| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| C1 | 响应式 | 在手机/平板/桌面端均可正常使用 |
| C2 | 加载状态 | 生成过程中按钮显示 loading，禁止重复点击 |
| C3 | 视觉一致 | 与现有落地页设计风格一致，但字体改用系统字体栈，保证离线可打开 |
| C4 | 文案友好 | 所有提示文案去技术词、简短、职场友好、清晰直接 |

## 10. 技术约束（供后续 TECH 阶段参考）

| 项 | 约束 |
|----|------|
| 前端 | 静态 HTML/CSS/JS，单页，无框架依赖（与现有 huihou-xingdongpai.html 一致） |
| 后端 | Node.js 轻量服务，仅做 AI 代理 + Key 保护 |
| AI 协议 | OpenAI 兼容协议（DashScope 兼容模式可用） |
| 部署 | 前端静态托管 + 后端轻量服务 |
| 设计风格 | 暖色系（#F5F4EF 底色 / #0D6E5C 主色），避免蓝紫配色 |
